# DESIGN_CODE.md — WB Shturval

> Единый источник правды для интерфейса. Все экраны обязаны соответствовать этому
> файлу. Токены ниже реализованы в `frontend/src/styles/tokens.css` один-в-один.
> Если дизайн-код меняется — правим этот файл и токены **вместе**, с записью в
> `CHANGELOG.md`.

## 1. Продукт и принципы

**WB Shturval** — рабочий кабинет снабжения для продавца Wildberries. Не лендинг,
не маркетинг. Инструмент, в котором сидят каждый день.

Принципы интерфейса:
1. **Сначала действие, потом аналитика.** Сразу видно, что делать сегодня.
2. **Числа — главный контент.** Таблицы плотные, цифры моноширинные и выровнены.
3. **Спокойствие.** Нейтральный фон, цвет — только для смысла (статусы, тренды).
4. **Честность состояния.** Загрузка, ошибка, неполные данные — всегда видны.
5. **Только светлая тема.** Тёмной темы пока нет (но токены вынесены, чтобы её
   можно было добавить позже без переписывания компонентов).

### Семантические (статусы и сигналы)

Каждый цвет имеет «насыщенный» (текст/иконка/граница) и «мягкий» (фон бейджа).

| Смысл              | Насыщенный | Мягкий фон | Где используется                         |
|--------------------|-----------|-----------|-------------------------------------------|
| Опасно / нет остатка (`danger`) | `#B42318` | `#FEECEB` | «Нет остатка», просрочка, ошибка        |
| Критично (`critical`)           | `#C4320A` | `#FFF1E8` | статус «Критично»                       |
| Внимание / заказать (`warning`) | `#B54708` | `#FEF3E2` | «Заказать», «Задерживается», неполные данные |

| Инфо / в пути (`info`)          | `#1D4ED8` | `#E8EFFE` | «В пути», «Норма синхронизации»         |

**Правило:** цвет статуса определяется **только** в одном месте
(`frontend/src/lib/status.ts` ↔ бэкенд-енам). Компоненты не зашивают цвета руками.

### Тренды чисел
- Рост/прибыль: `--success`. Падение/убыток: `--danger`. Ноль/нейтрально: `--text-2`.

## 4. Сетка, отступы, радиусы, тени

- **Шкала отступов (4px-базис):** `4, 8, 12, 16, 20, 24, 32, 40, 48`.
  Токены: `--sp-1`=4 … `--sp-6`=24, `--sp-8`=32 и т.д.
- **Контейнер контента:** max-width `1280px`, боковые поля `24px` (desktop).
- **Sidebar:** ширина `240px`, фиксированный; контент справа.
- **Радиусы:** `--r-sm`=6, `--r-md`=10, `--r-lg`=14, `--r-pill`=999.
- **Тени (мягкие, без «дизайнерского шума»):**
  - `--shadow-sm`: `0 1px 2px rgba(15,23,42,.06)`
  - `--shadow-md`: `0 4px 12px rgba(15,23,42,.08)`
  - `--shadow-pop`: `0 12px 32px rgba(15,23,42,.14)` (модалки, поповеры)
- **Граница вместо тени** — предпочтительна для статичных карточек/таблиц.

## 5. Layout приложения

```
┌──────────┬──────────────────────────────────────────────┐
│          │  Topbar: заголовок экрана · статус синка · ▢  │
│ Sidebar  ├──────────────────────────────────────────────┤
│ (логотип │                                              │
│  + меню) │   Контент экрана (max-width 1280, паддинг 24) │
│          │                                              │
└──────────┴──────────────────────────────────────────────┘
```

- **Sidebar-меню:** Дашборд · Товары · В пути · Настройки · (низ) Выход.
  Активный пункт: фон `--brand-soft`, текст `--brand`, левая полоска 3px `--brand`.
- **Topbar:** слева заголовок текущего экрана; справа — индикатор синхронизации
  (последняя синхронизация / «идёт…») и кнопка «Синхронизировать».

## 6. Таблицы (ключевой компонент)

Плотные, читаемые, числа справа.

- Высота строки: `44px` (плотный режим `--fs-dense`). Паддинг ячейки `8px 12px`.
- Шапка: фон `--surface-2`, текст `--text-2`, вес 600, размер 12, `UPPERCASE`
  опционально для коротких заголовков; «липкая» (`position: sticky; top: 0`).
- Разделители: только горизонтальные, `1px --border`. Вертикальных линий нет.
- Hover строки: фон `--surface-2`. Выбранная строка: фон `--brand-soft`.
- **Выравнивание:** текст слева, **числа справа**, статусы — бейджем.
- Числовые ячейки — `tabular-nums`, моноширинный токен.
- Сортируемый заголовок: курсор-pointer, стрелка ▲/▼ справа, активная — `--text`.
- Пустые числовые значения: `—` цветом `--text-3` (не `0`, если значение неизвестно).
- Чекбоксы массового выбора — первая колонка, фикс. ширина 40px.
- Адаптив: на узких экранах второстепенные колонки скрываются (см. §13), таблица
  горизонтально скроллится внутри карточки, ключевая колонка (артикул) — «липкая» слева.

## 7. Карточки и модальные окна

**Карточка:** фон `--surface`, граница `1px --border`, радиус `--r-lg`, паддинг
`16–20px`, заголовок `--fs-h2`. Тень только при «плавающем» состоянии.

**Модальное окно:**
- Оверлей: `rgba(15,23,42,.45)`.
- Окно: `--surface`, радиус `--r-lg`, тень `--shadow-pop`, max-width по контенту
  (обычно `480–640px`), паддинг `20–24px`.
- Шапка: заголовок слева, «×» справа. Футер: действия справа, главное — последним.
- Закрытие по «×», клику на оверлей и `Esc`. Фокус-ловушка внутри.

## 8. Кнопки и формы

### Кнопки
| Вариант     | Фон            | Текст        | Граница        | Применение            |
|-------------|----------------|--------------|----------------|-----------------------|
| `primary`   | `--brand`      | `#fff`       | —              | главное действие      |
| `secondary` | `--surface`    | `--text`     | `1px --border-strong` | вторичное      |
| `ghost`     | прозрачный     | `--text-2`   | —              | третичное, в строках   |
| `danger`    | `--danger`     | `#fff`       | —              | удаление/опасное       |

- Высоты: `sm`=32, `md`=38, `lg`=44. Радиус `--r-md`. Вес текста 600.
- Состояния: hover (затемнение акцента), active, disabled (`opacity .5`, без hover),
  loading (спиннер + блокировка). Фокус — кольцо `--brand-ring` (3px).

### Поля ввода
- Высота `md`=38, паддинг `8px 12px`, граница `1px --border-strong`, радиус `--r-md`,
  фон `--surface`. Фокус: граница `--brand`, кольцо `--brand-ring`.
- Лейбл сверху (`--fs-caption`, `--text-2`). Подсказка/ошибка снизу.
- Ошибка валидации: граница `--danger`, текст ошибки `--danger`, `--fs-caption`.
- Числовые поля (количество, цена) — выравнивание текста вправо, `inputmode=numeric`.
- Денежные значения форматируются как `1 234,56` (пробел-разделитель тысяч, запятая).

## 9. Статусы (бейджи)

Бейдж: `--r-pill`, паддинг `2px 10px`, `--fs-caption` (12/16, вес 600), мягкий фон +
насыщенный текст из таблицы §2. Слева может быть точка-индикатор 6px.

Снабжение:
- **Нет остатка** → `danger`
- **Критично** → `critical`
- **Заказать** → `warning`
- **Норма** → `success`
- **Избыток** → `neutral-strong`

Поставки:
- **В пути** → `info`
- **Частично приехала** → `info` (мягче) / точка `warning`
- **Задерживается** → `warning`
- **Остаток 0, поставка не обнаружена** → `danger`
- **Ждём после нуля** → `neutral-strong`
- **Доставлена** → `success`
- **Отменена** → `neutral` (приглушённый, `--text-3`)

Качество данных (финансы):
- **Проверенные** → `success`
- **Проверенные + буфер** → `info`
- **Частичная оценка** → `warning`
- **Нет данных** → `neutral`/`danger` (контекстно)

## 10. Графики

- Библиотека: Recharts (SVG, легко темизируется токенами).
- Сетка: горизонтальные линии `--border`, без вертикальных. Оси — `--text-3`,
  `--fs-caption`. Без 3D, без теней, без градиентной заливки «ради красоты».
- Цвета серий: выручка — `--brand`; прибыль — `--success`; убыток-зоны — `--danger`;
  продажи (шт) — `--info`.
- **Проверенные периоды vs прогнозный буфер различаются визуально:** проверенное —
  сплошная линия/полная заливка; текущая неделя/прогноз — **штриховка/пунктир** и
  пониженная непрозрачность, с пометкой в легенде «прогноз».
- Тултип: карточка `--surface`, тень `--shadow-pop`, числа `tabular-nums`.
- Пустой график: показываем заглушку §11, не пустой прямоугольник.

## 11. Пустые состояния, загрузка, ошибки

**Пустое состояние:** по центру блока — короткий заголовок (`--fs-h3`), пояснение
(`--text-2`) и одно действие (кнопка). Иконка нейтральная, без иллюстраций-«китов».
Пример первого запуска: «Нет данных. Подключите API-ключ Wildberries и запустите
синхронизацию» + кнопка «Открыть настройки».

**Загрузка:**
- Точечные данные → **skeleton** (серые блоки `--surface-2` с лёгким shimmer).
- Действие (кнопка) → инлайн-спиннер в кнопке.
- Долгая фоновая операция (синхронизация) → прогресс-индикатор с текущим шагом.

**Ошибка:** карточка с `danger`-акцентом: что случилось (человеческим языком),
что проверить/сделать, и кнопка «Повторить». **Никаких немых пустых экранов** —
ошибка всегда объясняется. Технические детали — под «Подробнее».

**Диагностические состояния** (обязательны, спец-раздел 0.4): ошибка синхронизации,
неполные данные, нет источника, неверный ключ, превышение лимитов WB, пустая
выгрузка, устаревшие данные, неполная финмодель, нет себестоимости — каждое имеет
узнаваемый бейдж/баннер и ведёт к действию.

## 12. Тосты (уведомления)

Правый нижний угол. Типы: success / warning / danger / info (цвета §2). Автоскрытие
4–6 c, кроме ошибок (закрываются вручную). Текст краткий + опц. действие («Отменить»,
«Открыть»). Не больше 3 одновременно (очередь).

## 13. Адаптив

Брейкпоинты: `--bp-sm`=640, `--bp-md`=960, `--bp-lg`=1280.
- ≥1280: sidebar + полный контент.
- 960–1279: sidebar сворачивается в иконки (ширина 64px), всплывающие подписи.
- <960: sidebar превращается в верхнее меню/бургер; таблицы скроллятся по горизонтали,
  ключевая колонка липкая; метрики — в 1–2 колонки.
- Минимальная цель касания: 40×40px. Целевая ширина проектирования — desktop-first
  (это рабочий кабинет), но без поломки на планшете/узком ноуте.

## 14. Доступность

- Контраст текста к фону ≥ 4.5:1 (проверено для `--text`/`--text-2` на `--surface`).
- Цвет **не единственный** носитель смысла: статусы дублируются текстом/иконкой.
- Фокус виден всегда (кольцо `--brand-ring`). Навигация с клавиатуры в модалках,
  меню, таблицах действий. `aria-label` на иконочных кнопках.

## 15. Библиотека базовых компонентов

Реализуются в `frontend/src/components/ui/` и используются везде (никакого
«локального» переопределения стилей):

`Button`, `IconButton`, `Input`, `NumberInput`, `Select`, `Textarea`, `Checkbox`,
`Toggle`, `Field` (лейбл+ошибка), `Card`, `Modal`, `Drawer`, `Table` (+ `Th` с сортировкой),
`StatusBadge`, `Metric` (метрика+подпись+тренд), `Tabs`, `Toast`/`ToastHost`,
`EmptyState`, `Skeleton`, `Spinner`, `Banner` (инфо/ошибка/предупреждение),
`Money`/`Num` (форматирование чисел), `ProgressSteps` (шаги синхронизации),
`DataQualityBadge`, `SyncStatus`.

**Правила компонентов:**
- Все цвета/размеры — только из токенов. Хардкод HEX в компонентах запрещён.
- Форматирование чисел/денег/дат — только через `Money`/`Num`/`formatDate`
  (единый источник, как формулы — один источник правды).
- Любой новый экран собирается из этих компонентов; если нужен новый паттерн —
  сначала добавить в библиотеку и в этот файл, потом использовать.

## 16. Тон текста (микрокопия)

- По-деловому, коротко, по-русски. Глагол действия в кнопках («Создать поставку»,
  «Запустить синхронизацию»).
- Числа — с единицами («12 шт», «8 дн», «1 240 ₽»). Валюта — рубль `₽` после числа.
- Ошибки — без вины пользователя и без жаргона: что случилось и что сделать.
