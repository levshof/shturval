# WB Shturval

Рабочий кабинет снабжения для продавца Wildberries: остатки, спрос, поставки,
закупки и честная экономика по артикулам. Каждый день показывает, **что заказать,
что принять, что рискует уйти в дефицит** и **какая реальная прибыль** с учётом
себестоимости, расходов WB, рекламы и налога.

> Главные приоритеты проекта: **надёжность, простота поддержки, корректность
> расчётов.** Без скрытых костылей и тихих подмен данных.

## Содержание репозитория

| Путь / файл                    | Назначение                                              |
|--------------------------------|---------------------------------------------------------|
| `backend/`                     | API-сервер (Node + TypeScript + Fastify + Prisma)       |
| `frontend/`                    | Веб-кабинет (React + TypeScript + Vite)                 |
| `WILDBERRIES_API_NOTES.md`     | Конспект актуального WB API (источник правды интеграции) |
| `DESIGN_CODE.md`               | Дизайн-код (светлая тема, токены) — обязателен для UI     |
| `DECISIONS.md`                 | Архитектурные/продуктовые решения и компромиссы          |
| `CHANGELOG.md`                 | Изменения по версиям                                    |
| `BUGFIX_LOG.md`                | Журнал багов: причина → фикс → проверка                  |
| `SETUP_GUIDE.md`               | Пошаговая настройка и запуск (для владельца)             |
| `docker-compose.yml`           | Локальный запуск (Postgres + backend + frontend)         |

## Архитектура (коротко)

```
┌────────────┐    HTTPS / JSON    ┌──────────────────────────┐   SQL   ┌────────────┐
│  Frontend  │ ◀───────────────▶ │  Backend (Fastify)        │ ◀────▶ │ PostgreSQL │
│  React SPA │                    │  ├─ auth (JWT)            │         └────────────┘
└────────────┘                    │  ├─ routes (REST)         │
                                  │  ├─ domain (формулы) ◀── единый источник правды
                                  │  ├─ wb client (rate-limit)│  1 req/min
                                  │  └─ sync engine (+guards) │ ───▶ Wildberries API
                                  └──────────────────────────┘
```

Слои строго разделены (спец-раздел 0.4): **загрузка данных** (wb/sync) ≠
**бизнес-расчёты** (domain, чистые функции) ≠ **отображение** (frontend). Формулы
снабжения и финансов живут в `backend/src/domain/` ровно в одном месте и покрыты
тестами.

## Быстрый старт (локально)

Подробно — в [`SETUP_GUIDE.md`](SETUP_GUIDE.md). Кратко:

```bash
# 1. Поднять PostgreSQL (через Docker)
docker compose up -d db

# 2. Backend
cd backend
cp .env.example .env          # заполнить значения (см. SETUP_GUIDE)
npm install
npm run db:migrate            # применить миграции Prisma
npm run dev                   # http://localhost:4000  (GET /health)

# 3. Frontend (в другом терминале)
cd frontend
cp .env.example .env          # указать VITE_API_URL
npm install
npm run dev                   # http://localhost:5173
```

Затем: зарегистрироваться → Настройки → вставить WB API-ключ (read-only, категории
«Статистика» + «Контент» + «Продвижение») → «Запустить синхронизацию».

## Технологии

- **Backend:** Node 24, TypeScript, Fastify, Prisma, PostgreSQL, Zod, Vitest.
- **Frontend:** React 18, TypeScript, Vite, React Router, TanStack Query, Recharts.
- **Интеграция:** Wildberries Statistics / Content / Promotion API
  (см. `WILDBERRIES_API_NOTES.md`).

## Принципы сопровождения

- Баги — через поиск **настоящей причины**, фикс фиксируется в `BUGFIX_LOG.md`.
- Любой временный обход — явно помечен и занесён в тех-долг (`DECISIONS.md`).
- Одна бизнес-формула — один источник правды (`backend/src/domain/`).
- Диагностические состояния (ошибка синка, неполные данные, нет ключа, лимиты WB,
  нет себестоимости) видны пользователю и в логах, а не прячутся.
