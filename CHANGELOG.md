# CHANGELOG — WB Shturval

Все заметные изменения по версиям. Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/),
версии — по [SemVer](https://semver.org/lang/ru/). Новые версии — сверху.

## [0.1.0] — 2026-06-30 — базовое ядро

Первая рабочая версия. Реализовано надёжное базовое ядро (спец-раздел 0.1).

### Added — документация-фундамент
- `WILDBERRIES_API_NOTES.md` — конспект актуального WB API (включая миграцию
  рекламы `adv/v2 → adv/v3` от 23.10.2025).
- `DESIGN_CODE.md` — дизайн-код (светлая тема, токены), `DECISIONS.md`,
  `CHANGELOG.md`, `BUGFIX_LOG.md`, `SETUP_GUIDE.md`.

### Added — backend (Node + TypeScript + Fastify + Prisma + PostgreSQL)
- Авторизация: регистрация/вход (email+пароль, bcrypt, JWT в httpOnly-cookie).
- Подключение WB-ключа с шифрованием AES-256-GCM и онлайн-проверкой.
- Доменное ядро формул (единый источник правды) + **80 unit-тестов**: доступные
  дни, скорость V7/V14/V30, взвешенная средняя, запас в днях, рекомендация заказа,
  избыток, статусы снабжения; экономика (себестоимость, выручка с fallback,
  расходы WB, реклама, налог, прибыль, маржа, упущенная прибыль, качество данных);
  прогноз текущей недели с уровнем уверенности; учёт поставок в пути.
- WB-клиент: rate-limiter (~1 запрос/мин), ретраи на 429/5xx, защита от
  бесконечной пагинации (orders/sales/report/cards/adv v3).
- Движок синхронизации: фоновый, идемпотентный, с защитой от публикации
  подозрительно неполной выгрузки (guards), статусом и сбросом зависаний.
- REST API: профиль, настройки, WB-ключ, товары (список/карточка/параметры/архив/
  импорт себестоимости), поставки, дашборд, скрытые задачи, синхронизация.

### Added — frontend (React + TypeScript + Vite)
- Дизайн-система по `DESIGN_CODE.md` (токены + базовые UI-компоненты), светлая тема.
- Экраны: вход, дашборд (финсводка, график, план на неделю, скрытые задачи, топ по
  прибыли, первый запуск), товары (фильтры/сортировка/поиск/массовые действия/
  импорт себестоимости), карточка товара (KPI, график продаж/финансов, экономика,
  поставка, параметры), «В пути», настройки.

### Added — деплой
- `docker-compose.yml`, Dockerfile backend/frontend, nginx-прокси, `.env.example`,
  воспроизводимая Prisma-миграция `init`.
- Бесплатный облачный вариант: `render.yaml` (Render blueprint), `frontend/vercel.json`
  (прокси `/api`), конфиг-ключ `COOKIE_SAMESITE` для same-origin / cross-site;
  пошаговый гайд в `SETUP_GUIDE.md` §20.11 (Supabase + Render + Vercel).

### Notes
- Тёмная тема намеренно не реализуется (требование ТЗ).
- Старый проект — только продуктовый референс; код/архитектура/БД не переносятся.
- Полный список отступлений от ТЗ — в `SETUP_GUIDE.md` §20.9 и `DECISIONS.md`.
